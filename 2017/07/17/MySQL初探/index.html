MySQL初探

发表于 2017-07-17 | 分类于 MySQL

MySQL 基本操作

MySQL语句规范

  • 关键字与函数名称全部大写
  • 数据库名称、表名称、字段名称全部小写
  • SQL语句必须以分号结尾

数据库操作

创建数据库

{ }必选项 | 表示从中作选择 [ ]可选可无

  • CREATE {DATABASE | SCHEMA} [IF NOT EXISTS] db_name [DEFAULT] CHARACTER SET [=] charset_name
    其中，[DEFAULT] CHARACTER SET [=] charset_name 表示创建的数据库的编码方式，[IF NOT EXISTS] 可让创建与数据库中的已有数据表名字相同时，不报错。
    如何查看当前数据库列表
  • SHOW {DATABASES | SCHEMAS}
    修改数据库
  • ALTER DATABASE
    删除数据库
  • DROP { DATABASE | SCHEMA } { IF EXISTS } db_name
    打开数据库
  • USE 数据库名称
    显示当前打开的数据库
  • SELECT DATABASE( )

数据类型与操作数据表

MySQL数据类型

定义：数据类型是指列、存储参数、表达式和具备变量的数据特征，她决定了数据的存储格式，代表了不同的信息格式。

整型

TINYINT 、SMALLINT、MEDIUMINT、INT、BIGINT 字节分别为1、2、3、4、8

浮点型

FLOAT[(M,D)] 、DOUBLE[(M,D)] M是数字总位数，D是小数点后面的位数。

日期时间型

YEAR 、TIME、DATA、DATETIME 、TIMESTAMP

字符型

CHAR、VARCHAR、TINYTEXT、TEXT、MEDIUMTEXT、LONGTEXT、ENUM(‘value1’,’value2’,…)枚举型、SET(‘value1,’value2’,…)最多64个成员

创建数据表

CREATE TABLE table_name(
column_name data_type， // column_name列名称，data_type 数据类型，逗号是两个字段之间的分隔符，最后一个不要分隔符
….
)

查看数据表

SHOW TABLES [FROM db_name] [LIKE ‘PATTERN’ | WHERE expr]

查看数据表结构

SHOW COLUMNS FROM tbl_name

MySQL记录的插入与查找

插入记录 //MySQL中行为记录

INSERT [INTO] tbl_name [(col_name,…)] VALUES(val,…)

记录查找

SELECT expr,…. FROM tbl_name
SELECT FROM tbl_name // 表示字段的筛选

MySQL的空值与非空

  • NULL 字段值可以为空
  • NOT NULL, 字段值禁止为空

    AUTO_INCREMENT

  • 自动编号，必须与主键组合使用
  • 默认情况下，起始值为1，每次增量为1

    主键（PRIMARY KEY）

  • 主键约束
  • 每张数据表只能存在一个主键
  • 主键保证记录的唯一性
  • 主键自动为NOT NULL

唯一约束（UNIQUE KEY）

  • 唯一约束
  • 可保证记录的唯一性
  • 字段可以存在空值（NULl）
  • 每张数据表可以存在多个唯一约束

    默认约束

  • 默认值
  • 当插入记录时，如果没有明确为字段赋值，则自动赋予默认值

约束以及修改数据表

外键约束的要求解析

约束

  • 约束保证数据的完整性以及一致性
  • 约束分为表级约束和列级约束
  • 约束类型包括：
    NOT NULL（非空约束）
    PRIMARY KEY （主键约束）
    UNIQUE KEY （唯一约束）
    DEFAULT （默认约束）
    FOREIGN KEY （外键约束）
    外键约束
  • 保持数据一致性，完整性
  • 实现一对一或一对多关系
    外键约束的要求

    子表：具有外键列的字表
    父表：子表参照的表
    子表和父表必须有相似的存储引擎

    编辑数据表的默认存储引擎

    default-storage-engine=INNODB
